# 기능 계획 가이드

이 디렉토리는 프로젝트의 기능 구현 계획 문서를 관리합니다. 모든 새로운 기능 개발은 TDD(Test-Driven Development) 원칙에 따라 단계별로 계획하고 구현합니다.

## 📋 계획 프로세스

### 1. 계획 문서 생성

새로운 기능을 구현하기 전에 `plan-template.md`를 기반으로 계획 문서를 생성합니다:

```bash
# 계획 문서는 다음 형식으로 생성합니다
docs/plans/PLAN_<feature-name>.md
```

### 2. 계획 작성 가이드

계획 문서 작성 시 다음 가이드라인을 따릅니다:

- **SKILL.md** 참조: Feature Planner 가이드라인 준수
- **TDD 원칙**: 테스트를 먼저 작성하고 구현
- **단계별 분해**: 각 단계는 1-4시간 내에 완료 가능하도록 분해
- **품질 게이트**: 각 단계 완료 후 품질 검증 필수

### 3. 계획 문서 구조

각 계획 문서는 다음 섹션을 포함합니다:

- **Overview**: 기능 설명, 성공 기준, 사용자 영향
- **Architecture Decisions**: 아키텍처 결정 및 근거
- **Dependencies**: 필요한 의존성
- **Test Strategy**: 테스트 전략 및 커버리지 목표
- **Implementation Phases**: 단계별 구현 계획 (RED-GREEN-REFACTOR)
- **Quality Gates**: 각 단계별 품질 검증 체크리스트
- **Risk Assessment**: 위험 평가 및 완화 전략
- **Rollback Strategy**: 롤백 전략

### 4. TDD 워크플로우

각 단계는 다음 순서로 진행됩니다:

1. **🔴 RED**: 실패하는 테스트 작성
2. **🟢 GREEN**: 테스트를 통과하는 최소 코드 작성
3. **🔵 REFACTOR**: 코드 품질 개선 (테스트는 계속 통과)

### 5. 품질 게이트

각 단계 완료 후 다음 항목을 검증합니다:

- **TDD 준수**: 테스트가 먼저 작성되었는지 확인
- **빌드 및 테스트**: 프로젝트 빌드 및 모든 테스트 통과
- **코드 품질**: 린팅, 타입 체크, 포맷팅 통과
- **보안 및 성능**: 보안 취약점 없음, 성능 저하 없음
- **문서화**: 코드 주석 및 문서 업데이트
- **수동 테스트**: 기능이 예상대로 작동하는지 확인

## 📚 템플릿 및 가이드

- **plan-template.md**: 계획 문서 템플릿 (프로젝트 루트)
- **SKILL.md**: Feature Planner 가이드라인 (프로젝트 루트)

## 📝 기존 계획 문서

- `PLAN_UI_and_Admin_System_Implementation.md`: UI 개선 및 관리자 시스템 구현 (완료)

## 🚀 새 기능 계획 시작하기

1. `plan-template.md`를 복사하여 `docs/plans/PLAN_<feature-name>.md` 생성
2. SKILL.md의 가이드라인에 따라 계획 작성
3. 사용자 승인 후 구현 시작
4. 각 단계별로 품질 게이트 통과 확인
5. 완료 후 계획 문서 업데이트

## ⚠️ 중요 사항

- **품질 게이트를 건너뛰지 마세요**: 각 단계의 모든 검증 항목을 통과해야 다음 단계로 진행할 수 있습니다.
- **테스트 우선**: 항상 테스트를 먼저 작성하고 구현합니다.
- **단계별 커밋**: 각 단계 완료 후 Git 커밋을 수행합니다.
